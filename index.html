§ 2.3 – 2.4

Hypothesis Testing & Correlation

🎯

2.3.2 P-Value & Confidence Intervals

A Confidence Interval (CI) is a range of values within which the true population parameter is expected to lie, at a given confidence level (e.g., 95%).

95% CI means: if we repeated the experiment 100 times, ~95 of the intervals would contain the true value.
🔬

2.3.3 Z-Test vs T-Test

Z-Test — large sample (n > 30), population variance known.

T-Test — small sample, population variance unknown.

Both tests compare means to check if there is a statistically significant difference between groups.

2.3.4 Chi-Square Test

Used to determine if there is a significant association between two categorical variables.

Example: Testing if gender and preferred product type are related.

2.3.5 ANOVA (Analysis of Variance)

Used when comparing means across three or more groups to see if at least one group mean is different from the others.

  • ▸ One-way: one independent variable
  • ▸ Two-way: two independent variables
  • ▸ Post-Hoc: Tukey's HSD to find which groups differ

2.4 Correlation & Covariance

Pearson Correlation

Measures strength & direction of a linear relationship between two continuous variables. Range: −1 to +1.

r = Σ(x−x̄)(y−ȳ) / √[Σ(x−x̄)²·Σ(y−ȳ)²]

Spearman Correlation

Non-parametric measure for ranked data. Assesses how well the relationship can be described by a monotonic function.

Correlation vs Causation

Correlation = two variables change together, but does NOT imply one causes the other.

Causation = one variable directly affects the other.

Always verify causal claims with experiments!

📊

Mean Imputation

Replace missing values with the average of that column. Best for normally distributed data without many outliers.

Python df['col'].fillna(df['col'].mean())
📈

Median Imputation

Use the middle value. Best for skewed data where mean would be distorted by extreme values.

Python df['col'].fillna(df['col'].median())
🏷️

Mode Imputation

Use the most frequent value. Ideal for categorical data like city names, product categories.

Python df['col'].fillna(df['col'].mode()[0])

Forward / Backward Filling

Used primarily in time-series data.

  • Forward Fill: Replace with the previous value
  • Backward Fill: Replace with the next available value

Dropping Missing Data

Remove rows or columns with missing values. Use only when missing data is minimal and won't affect results.

⚠️ Dropping too much data can introduce selection bias!
§ 4.2

Data Transformation

Scaling & Normalization

Standardization (Scaling): Brings all data to a similar scale (mean = 0, std = 1).

z = (x − μ) / σ

Normalization: Rescales data between 0 and 1. Useful for KNN, Neural Networks.

x_norm = (x − x_min) / (x_max − x_min)

Encoding Categorical Data

Converts non-numeric categories into numbers for ML models.

One-Hot Encoding: Creates binary columns.
Red → [1,0,0], Blue → [0,1,0], Green → [0,0,1]
Label Encoding: Converts to integers.
Red = 0, Blue = 1, Green = 2
§ 4.3

Handling Outliers

📉

Z-Score Method

Measures how many standard deviations a point is from the mean. Values above 3 or below −3 are usually outliers.

Z = (X − μ) / σ
📦

IQR Method

Interquartile Range method. Outliers fall below Q1 − 1.5×IQR or above Q3 + 1.5×IQR.

IQR = Q3 − Q1

Capping

Replace outlier values with the upper/lower boundary threshold value.

Removal

Delete rows containing outlier values when they represent erroneous data points.

Transformation

Apply log, square root, or Box-Cox transformations to reduce outlier impact.

6.3.3 Logistic Regression

Used when the output is binary (yes/no, 0/1). Output is a probability, transformed using the sigmoid curve.

P(y=1) = 1 / (1 + e^−(mx+b))
Use when: Predicting spam vs not-spam, churn vs no-churn, fraud vs legitimate.

6.4 ANOVA (Analysis of Variance)

One-way ANOVA

One independent variable across multiple groups.

Example: Test scores of students across 3 different schools.

Two-way ANOVA

Two independent variables examined simultaneously.

Example: Test scores across different schools AND teaching methods.

Post-Hoc Tests

Run after ANOVA to identify which specific groups differ.

Common test: Tukey's HSD (Honestly Significant Difference).

7.3.3 SVM (Support Vector Machines)

Finds the best boundary (hyperplane) that separates classes. Maximizes margin between support vectors.

Binary classification Text classification Image classification

7.3.4 KNN (K-Nearest Neighbors)

A "lazy learner" — stores all training data. For a new data point, finds the k closest points and uses majority voting (classification) or averaging (regression).

No model is trained; all computation happens during prediction.

7.3.5 Naive Bayes Classifier

Based on Bayes' Theorem. Assumes features are independent given the class label.

P(Class|Features) ∝ P(Features|Class) × P(Class)
Works well for text classification (e.g., spam detection).

§ 11

Machine Learning Landscape

Machine Learning helps data analysts build models that learn from data and make predictions or discover patterns. It's a core part of modern data analysis and business intelligence.

🎓 Supervised Learning

Train a model on labeled data — data that has input and the correct output.

  • ● Linear Regression — continuous output
  • ● Logistic Regression — binary output
  • ● Decision Trees — flowchart structure
  • ● Random Forest — ensemble approach
  • ● SVM — hyperplane classification
  • ● KNN — nearest neighbor

🔍 Unsupervised Learning

Used on unlabeled data. The model tries to find patterns or groupings without being told what to look for.

  • ● K-Means Clustering
  • ● DBSCAN (density-based)
  • ● PCA (dimensionality reduction)
  • ● t-SNE (visualization)

§ 13

Responsible Data Practice

🔐

Data Privacy & Security

Protecting individuals' personal data is both a legal requirement and an ethical obligation.

  • GDPR compliance for EU data subjects
  • Anonymization and pseudonymization
  • Minimal data collection principles
  • Ethical use of personal data
  • Right to be forgotten
⚖️

Bias in Data

Biased data leads to biased models that can perpetuate or amplify discrimination.

  • Identify sources of sampling bias
  • Test for historical bias in training data
  • Use fairness-aware ML techniques
  • Audit model outputs across groups
  • Document known limitations
🌍

Fairness in Models

Ensure AI decisions do not systematically disadvantage protected groups.

  • Demographic parity
  • Equalized odds / equal opportunity
  • Individual vs group fairness
  • Transparency & explainability
  • Stakeholder impact assessments

Introduction to Data Analysis

What it is, why it matters, and the four types every analyst must know

§ 1.1 – 1.3

What is Data Analysis?

Data Analysis is the process of examining, organizing, cleaning, and interpreting data to discover useful information, patterns, and trends. It helps individuals and organizations make informed decisions based on facts and evidence rather than guesses.

🛒 Simple Example: A shop owner looks at last month's sales records to see which product sold the most. That is a basic form of data analysis.
📊

1.2.1 Descriptive Analysis

Answers: "What happened?"

  • Focuses on summarizing historical data
  • Uses averages, percentages, charts, tables
  • 📌 Monthly report: 500 products sold in March
🔎

1.2.2 Diagnostic Analysis

Answers: "Why did it happen?"

  • Goes deeper to identify causes of outcomes
  • Compares variables or time periods
  • 📌 Sales dropped because website was down
🔮

1.2.3 Predictive Analysis

Answers: "What is likely to happen?"

  • Uses historical data to forecast future trends
  • Involves ML or statistical methods
  • 📌 Sales will rise during holiday season
💡

1.2.4 Prescriptive Analysis

Answers: "What should be done?"

  • Suggests actions based on data
  • Combines predictions + business rules
  • 📌 Recommend discount in low-performing region

§ 3.1 – 3.3

Types of Data & Data Sources

Data collection is the first and most crucial step. Understanding the type of data you're dealing with helps in choosing the right analysis method and tools.

3.1.1 Structured vs. Unstructured Data

🗃️ Structured Data
Organized in defined format (rows & columns). Easy to store in databases and analyze with standard tools.
Examples: Excel sheets, SQL databases, sales records.
📄 Unstructured Data
Does not follow a fixed format. More complex and harder to analyze.
Examples: Text, images, videos, social media posts, emails.

3.1.2 Qualitative vs. Quantitative Data

💬 Qualitative Data
Non-numerical. Describes qualities or characteristics. Usually collected through interviews or surveys.
Examples: Customer reviews, colors, emotions.
🔢 Quantitative Data
Numerical. Can be measured and analyzed statistically.
Examples: Age, income, test scores, temperatures.
🗄️

3.2.1 Databases

SQL (MySQL, PostgreSQL, Oracle) — Structured data in tables with fixed schemas.

NoSQL (MongoDB, Cassandra) — Flexible, semi-structured or unstructured data.

📂

3.2.2 Files & APIs

CSV/Excel — Common formats for tabular data.

JSON — Lightweight data-interchange format for web apps.

APIs — Fetch data from external sources in real time (weather, stock, government APIs).

🌐

3.2.3 Web Scraping

Extract data from websites using tools or scripts. Used when data is not available through APIs.

Dealing with Outliers — In Depth

Outliers are data significantly different from the rest of the dataset.

Example: In an employee salary dataset, if most salaries are $50–$200 but one entry shows $1,000 — that's an outlier. But it might be important! Maybe it reveals a manager earning unusually high.

Choosing the Right Statistical Test

A step-by-step decision framework from TileStats

§ Statistical Test Selection

5-Factor Decision Framework

The selection between statistical tests depends on five key factors.

❓

1. Your Question

Compare a difference between groups, or check a relationship between variables?

📏

2. Data Scale

Continuous (height, weight), Ordinal (rankings), or Nominal (categories)?

🔔

3. Normality

Are parametric test assumptions satisfied? (Normally distributed data?)

🔗

4. Study Design

Paired (same subject measured twice) or Unpaired (independent groups)?

👥

5. Number of Groups

One, two, or more than two groups being compared?

📐

Parametric vs Non-parametric

If data fairly normal → parametric. If skewed, n<30, or extreme values → use robust non-parametric tests.

Paired Study Design

Same subject measured twice — e.g., blood pressure before and after exercise, or control vs. drug on same individual.

Use Paired t-test (parametric) or Wilcoxon signed-rank (non-parametric)

Matched Pairs Design

Subjects matched on key characteristics (e.g., weight) and then randomly assigned to conditions.

Compare two groups that are very similar to reduce variability. One in control, one in treatment.

Parametric vs. Non-Parametric Models

📐 Parametric Models

Make assumptions regarding the data distribution and have parameters that are inferred from the data.

📌 Linear Regression is a parametric model — it assumes normality, linearity, homoskedasticity, no multicollinearity.

🌿 Non-Parametric Models

Have parameters related to the algorithm itself, but don't need assumptions about your data distribution.

📌 Random Forest is non-parametric — works on skewed data, doesn't require normality or scaling.

Descriptive Statistics — Deep Dive

Mean, Median, Mode, Variance, Standard Deviation, IQR with full formulas

§ 2.1

Measures of Central Tendency & Spread

μ

Mean (Average)

x̄ = (Σxᵢ) / N

Sum all values, divide by count. Represents the "central" value. Sensitive to outliers.

Example: [3,5,7] → mean = 15/3 = 5
M

Median (Middle)

Middle value when sorted

Less affected by extreme values (outliers) than the mean. Use with skewed data.

[1,3,7] → 3  |  [1,3,7,9] → (3+7)/2 = 5
Mo

Mode (Most Frequent)

Most repeated value

A dataset can have more than one mode, or no mode if all values are unique.

[2,4,4,6,7] → mode = 4

2.1.2 Variance & Standard Deviation

Variance = (1/N) Σ(xᵢ − μ)²

Measures how far data points are from the mean on average. Larger variance = more spread out.

SD (σ) = √Variance

SD is in the same units as the data. Small SD = clustered near mean. Large SD = widely spread.

2.1.3 Quartiles, Range & IQR

Q1
25th percentile
Q2
50th (median)
Q3
75th percentile
IQR
Q3 − Q1
IQR = Q3 − Q1

Range of the middle 50% of data. Robust measure — less sensitive to outliers.

Range: For [2,5,9] → 9 − 2 = 7

Probability Basics & Inferential Statistics

Probability rules, Bayes' Theorem, distributions, and hypothesis testing

§ 2.2 – 2.3

Probability Rules, Distributions & Inferential Stats

2.2.1 Basic Probability

P(A) = Favourable outcomes / Total outcomes

Sum of all possible outcomes = 1

Complement Rule:
P(Aᶜ) = 1 − P(A)

2.2.2 Conditional Probability

Probability of A occurring given that B has occurred:

P(A|B) = P(A ∩ B) / P(B)

Useful when events are dependent on each other.

2.2.3 Bayes' Theorem

Update probability of an event based on new information:

P(A|B) = P(B|A) × P(A) / P(B)

Used in medical testing, spam filtering, and machine learning (Naive Bayes).

2.2.4 Probability Distributions

Normal Distribution

Bell curve. Symmetric around the mean. Many natural phenomena follow this pattern.

Binomial Distribution

Models successes in fixed independent yes/no experiments with constant success probability.

Poisson Distribution

Events occurring in a fixed time/space interval at a constant average rate, independently.

2.3.1 Hypothesis Testing

H₀

Null Hypothesis (H₀)

A statement that there is no effect or difference. The default assumption.

H₁

Alternative Hypothesis (H₁)

A statement that there is an effect or difference. What we want to prove.

2.3.2 P-Value & Confidence Intervals

P-Value: The probability of obtaining observed results if H₀ were true.
If p < 0.05 → reject the null hypothesis
Confidence Interval: A range of plausible values for a population parameter. A 95% CI means if the study were repeated 100 times, 95 intervals would contain the true value.

🌲 Random Forest

Takes advantage of many decision trees, with rules created from subsamples of features. Each tree is weaker than a full decision tree, but combining them yields better overall performance — a wisdom of the crowd effect.

6.1.2 Type I and Type II Errors

⚠️
Type I Error
False Positive
Rejecting H₀ when it is actually true
α = significance level
🔕
Type II Error
False Negative
Failing to reject H₀ when it is actually false
β = probability
P < α (usually 0.05) → reject H₀. The significance level α controls the Type I error rate.

6.2 Confidence Intervals — Formulas

CI gives a range where we expect the true value to lie. A 95% CI means 95% confident the true value lies within that range.

For Mean:
CI = x̄ ± Z · (σ / √n)
Where x̄ = sample mean · Z = 1.96 (for 95%) · σ = standard deviation · n = sample size
For Proportions:
CI = p̂ ± Z · √(p̂(1 − p̂) / n)

Ethical Considerations — Full Chapter

§ 13.1 – 13.2

Data Ethics, Privacy & Fairness

Data can be powerful, but with power comes responsibility. Analysts must ensure data is handled fairly, securely, and without bias.

13.1.1 GDPR & Data Protection

GDPR (General Data Protection Regulation) is an EU law that protects personal data.

GDPR gives people rights to:

  • ✅ Know how their data is used
  • ✅ Delete their data ("right to be forgotten")
  • ✅ Consent before data is collected

If you collect data, you must:

  • 🔐 Get permission (consent)
  • 📣 Tell users why and how it's being used
  • 🛡️ Store it safely and protect from breaches
Example: Collecting email addresses for analysis → must tell users and encrypt the data.

13.1.2 Ethical Use of Personal Data

  • ❌ Never use personal data for purposes other than agreed
  • ❌ Avoid selling or sharing data without permission
  • 🔒 Anonymize sensitive info before using in reports/models
  • 📦 Follow "data minimization" — collect only what you need
Example: Analyzing customer purchases — you probably don't need their exact address or date of birth.

13.2.1 Identifying and Mitigating Bias

Data Bias

Data does not represent all groups fairly. Example: A health dataset with only male patients → biased conclusions for female patients.

Label Bias

The labels (outputs) used in supervised learning are flawed.

Confirmation Bias

Analysts interpret results to fit their expectations rather than the data.

How to Mitigate:

  • ✅ Use diverse datasets
  • ✅ Perform bias audits before model deployment
  • ✅ Involve domain experts when labeling data
  • ✅ Apply fairness metrics (Demographic Parity, Equal Opportunity)

13.2.2 Fairness in Data Analysis

Fairness means treating all groups equally when building models and making decisions.

✅ Test if model accuracy is similar across groups (genders, races, etc.)
❌ Avoid features that directly/indirectly relate to protected attributes (race, gender, religion)
🛠️ Use tools like IBM AI Fairness 360 to check model fairness
Real-life example: A hiring algorithm trained only on male CVs may reject qualified female applicants.

💡 Key Insight: Random Forest & Number of Features (datascienceharp)

Linear Regression caveat: Results can be compromised if you include too many variables. If enough junk variables are included, one of them is likely to meet the threshold for statistical significance just by chance — yet junk variables aren't always easy to recognize.
Random Forest advantage: Tends to be indifferent to the number of features you pass. With a large enough forest, junky combinations of features get eliminated on average, and the better-performing features get called to action to improve accuracy.

When the Difference question involves nominal data → use Chi-square test (unpaired) or Fisher's exact test (expected frequency <5). The Relationship branch + nominal variable also leads to Chi-square.

Stat Test Selection — Step 2: Scale of Measurement (TileStats)

After deciding Difference vs. Relationship, determine the scale of your measured variable(s).

Continuous

Measured on a numeric scale with infinite possible values.

Body weight, Blood pressure, Height, Temperature, BMI, Concentration

Ordinal

Ranked categories with an order, but intervals aren't equal.

Likert scale (1–5), Pain level, Education level

Nominal

Categories with no natural order.

Men/Women, Species A/B/C, Blood type, Smoking status
Key decision point: For continuous variables → check if parametric test assumptions (normality) are satisfied. If Yes → parametric tests (t-test, ANOVA, Pearson). If No → non-parametric tests (Wilcoxon, Mann-Whitney, Spearman).
